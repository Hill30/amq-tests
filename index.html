Case Study: Enterprise Integration with Mobile Applications

Introduction

Integrating enterprise hosted application with application deployed on mobile devices is challenging as it requires providing enterprise-level quality services over unstable networks with inconsistent bandwidth and security. We created this study to share knowledge obtain from real-life business deployment, to facilitate feedback and sharing of best practices, to put some extra emphasis on this and similar architectural solutions for open source products development to improve products’ stability and performance.

Tech Objectives

  1. Provide reliable communications with distributes network of mobile devices,
  2. Seamless integration of broad mobile network with internal messaging system.
  3. Support for most standard enterprise communication patterns:
    • Broadcasting and pair-to-pair communications.
    • Load-balancing and fault-tolerance
    • Message routing, filtering, etc...
    • Messages' expiration and escalation
    • Reliable message delivery
    • Secure communication.
  4. Utilize standard and best practices for communications with growing network of mobile devices:
    • Standard-based communication protocol
    • Supported on Android and iOS,
    • Non-reliable network connections,
    • Energy efficient communication patterns
    • Performance related:
      • Max concurrently connected mobile devices/clients – 10,000.
      • Max communication traffic: 1,000 messages 1Mb total per device per day.
      • The latency of the message delivery for connected client – 60 seconds.
  5. Minimized maintenance of messages
    • Undelivered messages to mobile devices should expire in 48 hours.
    • Dormant client subscriptions should be removed with all messages cleared in 10 days.

Methodology and Assumptions

  • Test environment will be used verify business objectives.
  • Testing will be done first for single broker and then for load-balanced deployment.
  • Test client will be created to simulate real-life mobile devices.
  • Network disconnect will be emulated by standard server firewall turn off-on.
  • Required solution’s security will be provided by standard products’ functionality

ActiveMQ Configuration.

  1. Added/enabled MQTT adapter to Active MQ configuration.
  2. Increased max connections to 10,000
  3. Changed communication protocol to NIO (greatly reduces number of active threads for large number connections).
  4. Enabled more memory for the broker - 4GB (this is maximum using default wrapper that comes with ActiveMQ installation package).
  5. Enabled thread pool.
  6. Added transport level security – SSL.
